  • Methodology

    • Geospatial Processing

      - Identify and define corridors
      - Segmentize network into 10-meter segments
      - Buffer and snap crash to nearest segment

    • Sliding Window Calculation

      - Apply a sliding window calculation to compute crash per mile values for each 10-meter segment
      - Use multiple window sizes to generate a range of estimates for each segment
      - Average these estimates to minimize bias and obtain a final KSI per mile value

    • Post-Processing

      - Utilize the composite KSI per mile for determining the inclusion of segments in the HIN
      - Post-process to fill in gaps, and remove corridors/sections below certain threshold
